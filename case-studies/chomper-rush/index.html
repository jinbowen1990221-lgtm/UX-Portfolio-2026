01 / OVERVIEW

分数既是奖励，
也是被围猎的理由

传统吃豆的目标是清场；这个原型把目标改成动态排名。玩家需要在“继续发育”和“主动攻击”之间不断选择。

核心设计张力：你越强，拾取范围和战斗收益越高，但高分也会吸引更多猎手，安全路线随排名实时变化。

产品移动端横屏竞技游戏
体验目标资源争夺、成长、反杀、冲榜
输入虚拟摇杆 + BOOST
技术边界原生 Canvas 单页原型

02 / CORE LOOP

让每一次收集，
都改变下一步风险

玩法不是“吃得越多越好”的单线成长，而是一条持续升压的反馈回路：发育带来能力，也把玩家暴露给更强的追击。

  1. 01
    收集

    豆子、金币与高价值资源点提供得分和成长。

  2. 02
    成长

    体型、拾取范围与战斗收益随成长值提升。

  3. 03
    武装

    能量豆与道具把追逐关系暂时反转。

  4. 04
    冲榜

    排名上升带来更高目标价值和悬赏收益。

  5. 05
    被围猎

    高分玩家触发更多 AI 猎手，迫使路线重新规划。

03 / PLAY EXPERIENCE

一个横屏里，同时看见操作、成长、威胁和排名

HUD 把高频判断放在画面上缘；排行榜保持在左侧；摇杆与 BOOST 分居两端，减少拇指遮挡中心战场。

01

拇指操作不遮战场

左右控制布局可在设置中互换，兼顾不同握持习惯；桌面端同步支持键盘。

02

状态常驻而非临时弹窗

分数、生命、连击、能量、成长、在线人数与排名保持可见，避免战斗中断读。

03

危险通过追逐关系表达

守卫、精英 AI 和高分猎手不是静态障碍，而是会根据玩家状态切换目标的对手。

04 / MODES

同一套核心循环，服务两种游戏节奏

无尽模式强调持续发育和生存；排位模式把时间压缩到 180 秒，让资源点、悬赏和击败时机更关键。

05 / SYSTEM DESIGN

AI 不只追你，
它们有不同生存策略

AI 每约 400ms 重算一次意图，在生存、争夺资源、伏击和发育之间选择。不同性格让场上压力不只来自速度差。

HUNTER

猎手

主动追击玩家和高分目标；拥有战斗资源时更倾向持续锁定。

FARMER

发育者

优先收集豆子和金币，靠稳定资源效率进入中后期竞争。

RAIDER

掠夺者

绕行获取能量豆或控制道具，再寻找伏击和反杀机会。

COWARD / ELITE

逃生者与精英

一类主动规避威胁，另一类制造高位排名压力，让战局保持层次。

06 / IMPLEMENTATION

用原生 Web 技术，把设计判断直接变成可玩的版本

没有框架和服务端依赖。当前版本聚焦横屏渲染、触控反馈、AI 决策、音效和本地进度，适合快速验证核心玩法。

  1. INPUTPointer / Keyboard

    摇杆、BOOST、WASD 与方向键

  2. STATEJavaScript Loop

    成长、道具、碰撞、排名与 AI 意图

  3. RENDERCanvas 2D

    迷宫、角色、资源、粒子与摄像机

  4. FEEDBACKWeb Audio

    程序化背景音乐和拾取、攻击音效

  5. MEMORYlocalStorage

    金币、段位、皮肤和操作设置
